# Integration

Clearance produces outputs in standard OpenStreetMap ecosystem formats, making integration with your existing tools straightforward.

## Standard outputs

### OSM diff files

Clearance produces diff files (`.osc`) in the standard OSM format. These files can be consumed by any tool compatible with the OSM replication format:

- **Imposm** — Import into PostgreSQL/PostGIS
- **osm2pgsql** — Import into PostgreSQL with custom schema
- **Osmosis** — OSM data processing

### Validated database

Clearance maintains a database that only contains validated changes. This database can be queried directly.

## Overpass API proxy

Clearance provides a **transparent Overpass API proxy**. Your applications that already query the Overpass API can point to the Clearance proxy without code changes.

The proxy:

- Intercepts Overpass queries
- Redirects them to the Clearance-validated database
- Returns results in the standard Overpass format

**Benefit**: your applications get Clearance-filtered data without any client-side modification.

## Integration architecture

```
OpenStreetMap
      ↓
  Replication
      ↓
  Clearance ──→ .osc files ──→ osm2pgsql / Imposm
      ↓
  Overpass proxy ──→ Your applications
```
