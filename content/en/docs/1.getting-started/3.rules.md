# Validation rules

Clearance includes seven categories of validation rules. Each rule can be enabled, disabled, and configured according to your needs.

## Delayed

Detects changes that arrive with a significant delay compared to the normal replication feed. A significant delay may indicate a problematic modification or a mass import.

## Deleted

Monitors deletions of OpenStreetMap objects. Deleting an object used by your applications can cause malfunctions. This rule allows you to control deletions before they take effect.

**Detected cases:**

- Deletion of nodes, ways, or relations
- Deletion of objects referenced by other objects

## Duplicate

Identifies object creations that appear to be duplicates of existing objects. Duplicates pollute the database and can create confusion.

**Detection criteria:**

- Geographic proximity to an existing object
- Tag similarity

## Geom (Geometry)

Checks the validity and consistency of geometric changes. Significant moves or invalid geometries are flagged.

**Detected cases:**

- Significant node displacement
- Invalid geometry (self-intersection, etc.)
- Way shape modification

## Network

Controls the consistency of the road network and public transport. Changes that break network connectivity are flagged.

**Detected cases:**

- Connectivity breaks in the road network
- Inconsistencies in transport relations

## Tags

Checks the semantic consistency of OSM tags. Tag changes that don't follow conventions are flagged.

**Detected cases:**

- Deletion of important tags
- Addition of inconsistent tags
- Modification of key tags (name, ref, etc.)

## User List

Allows defining lists of users whose changes require special attention. Can be used to monitor recently created accounts or contributors with a history of problematic edits.

**Modes:**

- **Whitelist** — Only listed users pass automatically
- **Blacklist** — Changes from listed users are systematically held
