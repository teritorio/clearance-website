# How Clearance works

## OSM replication under quality constraints

Clearance lets you implement a **partial OpenStreetMap replication under quality constraints**. Rather than blindly accepting all changes from the replication feed, Clearance analyzes them and only passes those that meet your criteria.

## Geographic context of changes (LoCha)

Clearance introduces the concept of **Logical Changeset** (LoCha).

In OpenStreetMap, a _changeset_ is a set of changes grouped by a single contributor during an editing session. However, a single changeset can contain edits in very different geographic areas.

LoCha groups changes by **geographic proximity** to create coherent clusters:

- Geographically close changes are grouped together
- Each LoCha represents a local change context
- The operator can evaluate changes in their spatial context

## Semantic analysis

Clearance performs semantic analysis of changes:

- **Tag verification** — Detection of inconsistent or missing tags
- **Before/after comparison** — Analysis of differences between previous and new state
- **Anomaly detection** — Identification of unusual modification patterns

## Validation flow

The validation flow proceeds in several steps:

### 1. Receiving changes

Clearance connects to the OpenStreetMap replication feed and fetches changes at regular intervals.

### 2. Applying rules

Each change is evaluated against all configured [validation rules](/en/docs/getting-started/rules). A change can trigger zero, one, or multiple rules.

### 3. Automatic decision

- **No rules triggered** → The change is automatically integrated
- **At least one rule triggered** → The change is held for review

### 4. Human review

The operator accesses the Clearance web interface to examine held changes. For each LoCha, they can:

- **Accept** — The change is integrated into the validated database
- **Reject** — The change is excluded

### 5. Output

Validated changes (automatically or manually) are made available as [standard outputs](/en/docs/going-further/integration).
