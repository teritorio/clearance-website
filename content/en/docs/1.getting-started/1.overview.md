# Overview

## What is Clearance?

Clearance is an **open source quality filter** for your OpenStreetMap replication feed. It sits in the replication pipeline, between the global OpenStreetMap database and your replication tools (osm2pgsql, Imposm, Osmosis), to filter and validate changes before they reach your systems.

## Reusing OpenStreetMap data with confidence

OpenStreetMap is an open geographic database used to create maps, power mobility services, enrich reference databases, and analyze territories. Data is published under the open **ODbL license**. This global digital commons is used by thousands of applications and is now a reference for many use cases.

When this data becomes critical for a service, a question arises: **how do you control the quality of data from OpenStreetMap?**

OpenStreetMap is constantly evolving. Every day, millions of changes are made by a global community of contributors: volunteers, public institutions, and companies. This dynamism is a strength, but errors, most often unintentional, can affect your services:

- **Accidental edits** — A well-intentioned contributor moves a point of interest by mistake
- **Inconsistencies** — Incorrect tags or invalid geometries
- **Unwanted deletions** — Objects deleted without valid reason
- **Vandalism** — Malicious edits that corrupt data

Classic OSM data replication tools do not include a quality filter. They operate on data that has already been replicated, or at the cost of blocking replication. Clearance solves this problem by establishing quality control **during** replication.

## How it works

Clearance never modifies OpenStreetMap. Problematic data is simply held, and corrections are made directly in OSM at the source.

```
OpenStreetMap → Replication → Clearance → Your applications
                                 ↓
                          Compliant changes → Pass automatically
                          Suspicious changes → Human review
```

1. **Replication** — Clearance connects to the OSM replication feed
2. **Analysis** — Each change is evaluated against validation rules
3. **Filtering** — Compliant changes are automatically integrated
4. **Review** — Suspicious changes are presented for human validation

You get an up-to-date version of OpenStreetMap data, reliable for your use cases and adapted to your quality constraints, while continuing to contribute to the OpenStreetMap commons.
