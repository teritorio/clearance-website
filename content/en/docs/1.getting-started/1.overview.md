# Overview

## What is Clearance?

Clearance is an **open source quality gate** for OpenStreetMap data. It sits between the global OpenStreetMap database and your applications to filter and validate changes before they reach your systems.

## Why Clearance exists

OpenStreetMap is an open collaborative database. Anyone can edit the data, which is its greatest strength — but also a challenge for organizations that depend on it.

Common problems:

- **Accidental edits** — A well-intentioned contributor moves a point of interest by mistake
- **Vandalism** — Malicious edits that corrupt data
- **Inconsistencies** — Incorrect tags or invalid geometries
- **Unwanted deletions** — Objects deleted without valid reason

Clearance solves these problems by establishing quality control between OpenStreetMap and your applications.

## How it works

```
OpenStreetMap → Replication → Clearance → Your applications
                                 ↓
                          Compliant changes → Pass automatically
                          Suspicious changes → Human review
```

1. **Replication** — Clearance connects to the OSM replication feed
2. **Analysis** — Each change is evaluated against validation rules
3. **Filtering** — Compliant changes are automatically integrated
4. **Review** — Suspicious changes are presented for human validation
