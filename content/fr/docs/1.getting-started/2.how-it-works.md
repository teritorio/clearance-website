# Comment fonctionne Clearance

## Une réplication OSM sous contraintes de qualité

Clearance vous permet de mettre en œuvre une **réplication partielle d'OpenStreetMap sous contraintes de qualité**. Plutôt que d'accepter aveuglément toutes les modifications du flux de réplication, Clearance les analyse et ne laisse passer que celles qui respectent vos critères.

## Contexte géographique des changements (LoCha)

Clearance introduit le concept de **groupe de changements logiques** (LoCha — _Logical Changeset_).

Dans OpenStreetMap, un _changeset_ est un ensemble de modifications regroupées par un même contributeur lors d'une session d'édition. Cependant, un seul changeset peut contenir des modifications dans des zones géographiques très différentes.

LoCha regroupe les modifications par **proximité géographique** pour créer des clusters cohérents :

- Les modifications proches géographiquement sont regroupées
- Chaque LoCha représente un contexte de changement local
- L'opérateur peut ainsi évaluer les modifications dans leur contexte spatial

## Analyse sémantique

Clearance effectue une analyse sémantique des modifications :

- **Vérification des tags** — Détection des tags incohérents ou manquants
- **Comparaison avant/après** — Analyse des différences entre l'état précédent et le nouvel état
- **Détection d'anomalies** — Identification des patterns de modification inhabituels

## Flux de validation

Le flux de validation se déroule en plusieurs étapes :

### 1. Réception des modifications

Clearance se connecte au flux de réplication OpenStreetMap et récupère les modifications à intervalles réguliers.

### 2. Application des règles

Chaque modification est évaluée par l'ensemble des [règles de validation](/fr/docs/getting-started/rules) configurées. Une modification peut déclencher zéro, une ou plusieurs règles.

### 3. Décision automatique

- **Aucune règle déclenchée** → La modification est automatiquement intégrée
- **Au moins une règle déclenchée** → La modification est retenue pour revue

### 4. Revue humaine

L'opérateur accède à l'interface web de Clearance pour examiner les modifications retenues. Pour chaque LoCha, il peut :

- **Accepter** — La modification est intégrée dans la base validée
- **Rejeter** — La modification est exclue

### 5. Sortie

Les modifications validées (automatiquement ou manuellement) sont mises à disposition sous forme de [sorties standards](/fr/docs/going-further/integration).
