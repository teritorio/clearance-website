# Vue d'ensemble

## Qu'est-ce que Clearance ?

Clearance est un **filtre qualité open source** pour les données OpenStreetMap. Il s'intercale entre la base de données mondiale OpenStreetMap et vos applications pour filtrer et valider les modifications avant qu'elles n'atteignent vos systèmes.

## Réutiliser les données OpenStreetMap avec confiance

OpenStreetMap est une base de données géographiques ouverte, utilisée pour créer des cartes, alimenter des services de mobilité, enrichir des référentiels ou analyser un territoire. Les données sont publiées sous licence libre **ODbL**. Ce commun numérique mondial est utilisé par des milliers d'applications et constitue aujourd'hui une référence pour de nombreux usages.

Lorsque ces données deviennent critiques pour un service, une question apparaît : **comment maîtriser la qualité des données issues d'OpenStreetMap ?**

OpenStreetMap évolue en permanence. Chaque jour, des millions de modifications sont apportées par une communauté mondiale de contributeurs : bénévoles, institutions publiques et entreprises. Cette dynamique est une force, mais des erreurs, le plus souvent involontaires, peuvent affecter vos services :

- **Modifications accidentelles** — Un contributeur bien intentionné déplace un point d'intérêt par erreur
- **Incohérences** — Des tags mal renseignés ou des géométries invalides
- **Suppressions non souhaitées** — Des objets supprimés sans raison valable
- **Vandalisme** — Des modifications malveillantes qui corrompent les données

Les outils classiques de réplication des données OSM n'intègrent pas de filtre qualité. Ils interviennent sur des données déjà répliquées, ou au prix d'un blocage de la réplication. Clearance résout ce problème en instaurant un contrôle qualité **pendant** la réplication.

## Principe de fonctionnement

Clearance ne modifie jamais OpenStreetMap. Les données problématiques sont simplement mises en attente, et les corrections sont apportées directement dans OSM à la source.

```
OpenStreetMap → Réplication → Clearance → Vos applications
                                 ↓
                          Modifications conformes → Passent automatiquement
                          Modifications suspectes → Revue humaine
```

1. **Réplication** — Clearance se connecte au flux de réplication OSM
2. **Analyse** — Chaque modification est évaluée par des règles de validation
3. **Filtrage** — Les modifications conformes sont intégrées automatiquement
4. **Revue** — Les modifications suspectes sont présentées pour validation humaine

Vous disposez ainsi d'une version à jour des données OpenStreetMap, fiable pour vos usages et adaptée à vos contraintes de qualité, tout en continuant à contribuer au commun OpenStreetMap.
