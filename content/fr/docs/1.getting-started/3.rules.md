# Règles de validation

Clearance dispose de sept catégories de règles de validation. Chaque règle peut être activée, désactivée et configurée selon vos besoins.

## Delayed (Retard)

Détecte les modifications qui arrivent avec un retard significatif par rapport au flux de réplication normal. Un retard important peut indiquer une modification problématique ou un import massif.

## Deleted (Suppression)

Surveille les suppressions d'objets OpenStreetMap. La suppression d'un objet utilisé par vos applications peut causer des dysfonctionnements. Cette règle permet de contrôler les suppressions avant qu'elles ne prennent effet.

**Cas détectés :**

- Suppression de nœuds, voies ou relations
- Suppression d'objets référencés par d'autres objets

## Duplicate (Doublon)

Identifie les créations d'objets qui semblent être des doublons d'objets existants. Les doublons polluent la base de données et peuvent créer de la confusion.

**Critères de détection :**

- Proximité géographique avec un objet existant
- Similarité des tags

## Geom (Géométrie)

Vérifie la validité et la cohérence des modifications géométriques. Les déplacements importants ou les géométries invalides sont signalés.

**Cas détectés :**

- Déplacement significatif d'un nœud
- Géométrie invalide (auto-intersection, etc.)
- Modification de la forme d'une voie

## Network (Réseau)

Contrôle la cohérence du réseau routier et des transports en commun. Les modifications qui cassent la connectivité du réseau sont signalées.

**Cas détectés :**

- Rupture de connectivité dans le réseau routier
- Incohérences dans les relations de transport

## Tags

Vérifie la cohérence sémantique des tags OSM. Les modifications de tags qui ne respectent pas les conventions sont signalées.

**Cas détectés :**

- Suppression de tags importants
- Ajout de tags incohérents
- Modification de tags clés (name, ref, etc.)

## User List (Liste d'utilisateurs)

Permet de définir des listes d'utilisateurs dont les modifications nécessitent une attention particulière. Peut être utilisé pour surveiller les comptes récemment créés ou les contributeurs ayant un historique de modifications problématiques.

**Modes :**

- **Liste blanche** — Seuls les utilisateurs listés passent automatiquement
- **Liste noire** — Les modifications des utilisateurs listés sont systématiquement retenues
