# Une boucle d'amélioration continue

Lorsqu'un changement ne passe pas les règles de qualité, il n'est pas perdu : il est **retenu pour relecture**.

Deux cas sont alors possibles :

## Correction dans OpenStreetMap

Une correction est apportée dans OpenStreetMap. À la prochaine mise à jour, si les objets concernés deviennent conformes, les changements cumulés sont **intégrés automatiquement** dans la base locale.

## Acceptation manuelle

La modification est jugée valide par l'utilisateur malgré le signalement. Elle est **acceptée manuellement** et intégrée à la base locale.

---

Cette approche permet de maintenir une copie OSM stable, dans laquelle la qualité des changements intégrés est contrôlée, **sans nécessiter de synchronisation complète**.
