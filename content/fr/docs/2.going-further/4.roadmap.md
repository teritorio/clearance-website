# Feuille de route

Clearance est en développement actif. Voici les grandes orientations pour les prochaines versions.

## Financement NLNet

Le développement de Clearance est soutenu par **NLNet**, dans le cadre du programme NGI (Next Generation Internet). Ce financement permet d'accélérer le développement de fonctionnalités clés.

## Sources de données externes

**Objectif** : Enrichir l'analyse de Clearance en intégrant des sources de données externes pour la comparaison et la validation.

- Comparaison avec des bases de données de référence (cadastre, bases adresses, etc.)
- Détection de régressions par rapport à des données officielles
- Validation croisée avec d'autres sources ouvertes

## Système de réputation

**Objectif** : Affiner le filtrage en tenant compte de l'historique et de la fiabilité des contributeurs.

- Score de confiance par contributeur basé sur l'historique
- Seuils de validation adaptatifs selon la réputation
- Apprentissage progressif à partir des décisions de validation

## Améliorations de l'interface

- Tableau de bord avec statistiques de validation
- Visualisation cartographique améliorée des LoCha
- Notifications et alertes configurables

## Contribuer

Clearance est open source. Les contributions sont les bienvenues sur le [dépôt GitHub](https://github.com/teritorio/clearance).

- Signaler un bug ou proposer une amélioration via les issues
- Soumettre une pull request
- Participer aux discussions
