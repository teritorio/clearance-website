# Intégration

Clearance produit des sorties dans des formats standards de l'écosystème OpenStreetMap, facilitant l'intégration avec vos outils existants.

## Sorties standards

### Fichiers de différences OSM

Clearance produit des fichiers de différences (`.osc`) au format standard OSM. Ces fichiers peuvent être consommés par tout outil compatible avec le format de réplication OSM :

- **Imposm** — Import dans PostgreSQL/PostGIS
- **osm2pgsql** — Import dans PostgreSQL avec schéma personnalisé
- **Osmosis** — Traitement de données OSM

### Base de données validée

Clearance maintient une base de données qui ne contient que les modifications validées. Cette base peut être interrogée directement.

## Proxy Overpass API

Clearance propose un **proxy Overpass API** transparent. Vos applications qui interrogent déjà l'Overpass API peuvent pointer vers le proxy Clearance sans modification de code.

Le proxy :

- Intercepte les requêtes Overpass
- Les redirige vers la base de données validée par Clearance
- Retourne les résultats dans le format Overpass standard

**Avantage** : vos applications obtiennent des données filtrées par Clearance sans aucune modification côté client.

## Architecture d'intégration

```
OpenStreetMap
      ↓
  Réplication
      ↓
  Clearance ──→ Fichiers .osc ──→ osm2pgsql / Imposm
      ↓
  Proxy Overpass ──→ Vos applications
```

## Configuration

Pour configurer l'intégration avec votre infrastructure, consultez le [dépôt GitHub](https://github.com/teritorio/clearance) pour les instructions détaillées d'installation et de configuration.
