# Cómo funciona Clearance

## Una replicación OSM bajo restricciones de calidad

Clearance le permite implementar una **replicación parcial de OpenStreetMap bajo restricciones de calidad**. En lugar de aceptar ciegamente todos los cambios del flujo de replicación, Clearance los analiza y solo deja pasar los que cumplen sus criterios.

## Contexto geográfico de los cambios (LoCha)

Clearance introduce el concepto de **grupo de cambios lógicos** (LoCha — _Logical Changeset_).

En OpenStreetMap, un _changeset_ es un conjunto de modificaciones agrupadas por un mismo contribuidor durante una sesión de edición. Sin embargo, un solo changeset puede contener ediciones en áreas geográficas muy diferentes.

LoCha agrupa los cambios por **proximidad geográfica** para crear clusters coherentes:

- Los cambios geográficamente cercanos se agrupan
- Cada LoCha representa un contexto de cambio local
- El operador puede evaluar los cambios en su contexto espacial

## Análisis semántico

Clearance realiza un análisis semántico de los cambios:

- **Verificación de tags** — Detección de tags inconsistentes o faltantes
- **Comparación antes/después** — Análisis de las diferencias entre el estado anterior y el nuevo
- **Detección de anomalías** — Identificación de patrones de modificación inusuales

## Flujo de validación

El flujo de validación se desarrolla en varias etapas:

### 1. Recepción de cambios

Clearance se conecta al flujo de replicación de OpenStreetMap y obtiene los cambios a intervalos regulares.

### 2. Aplicación de reglas

Cada cambio se evalúa contra todas las [reglas de validación](./rules) configuradas. Un cambio puede activar cero, una o varias reglas.

### 3. Decisión automática

- **Ninguna regla activada** → El cambio se integra automáticamente
- **Al menos una regla activada** → El cambio se retiene para revisión

### 4. Revisión humana

El operador accede a la interfaz web de Clearance para examinar los cambios retenidos. Para cada LoCha, puede:

- **Aceptar** — El cambio se integra en la base validada
- **Rechazar** — El cambio se excluye

### 5. Salida

Los cambios validados (automática o manualmente) se ponen a disposición como [salidas estándar](./integration).
