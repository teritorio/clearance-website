# Reglas de validación

Clearance dispone de siete categorías de reglas de validación. Cada regla puede activarse, desactivarse y configurarse según sus necesidades.

## Delayed (Retraso)

Detecta los cambios que llegan con un retraso significativo respecto al flujo de replicación normal. Un retraso importante puede indicar una modificación problemática o una importación masiva.

## Deleted (Eliminación)

Supervisa las eliminaciones de objetos OpenStreetMap. La eliminación de un objeto utilizado por sus aplicaciones puede causar mal funcionamiento. Esta regla permite controlar las eliminaciones antes de que surtan efecto.

**Casos detectados:**

- Eliminación de nodos, vías o relaciones
- Eliminación de objetos referenciados por otros objetos

## Duplicate (Duplicado)

Identifica las creaciones de objetos que parecen ser duplicados de objetos existentes. Los duplicados contaminan la base de datos y pueden crear confusión.

**Criterios de detección:**

- Proximidad geográfica con un objeto existente
- Similitud de tags

## Geom (Geometría)

Verifica la validez y coherencia de las modificaciones geométricas. Los desplazamientos importantes o las geometrías inválidas se señalan.

**Casos detectados:**

- Desplazamiento significativo de un nodo
- Geometría inválida (autointersección, etc.)
- Modificación de la forma de una vía

## Network (Red)

Controla la coherencia de la red viaria y del transporte público. Los cambios que rompen la conectividad de la red se señalan.

**Casos detectados:**

- Ruptura de conectividad en la red viaria
- Inconsistencias en las relaciones de transporte

## Tags

Verifica la coherencia semántica de los tags OSM. Los cambios de tags que no respetan las convenciones se señalan.

**Casos detectados:**

- Eliminación de tags importantes
- Adición de tags inconsistentes
- Modificación de tags clave (name, ref, etc.)

## User List (Lista de usuarios)

Permite definir listas de usuarios cuyas modificaciones requieren atención especial. Puede usarse para supervisar cuentas recién creadas o contribuidores con un historial de modificaciones problemáticas.

**Modos:**

- **Lista blanca** — Solo los usuarios listados pasan automáticamente
- **Lista negra** — Los cambios de los usuarios listados se retienen sistemáticamente
