# Descripción general

## ¿Qué es Clearance?

Clearance es un **filtro de calidad open source** para datos de OpenStreetMap. Se interpone entre la base de datos mundial de OpenStreetMap y sus aplicaciones para filtrar y validar los cambios antes de que lleguen a sus sistemas.

## Reutilizar los datos de OpenStreetMap con confianza

OpenStreetMap es una base de datos geográfica abierta, utilizada para crear mapas, alimentar servicios de movilidad, enriquecer bases de datos de referencia y analizar territorios. Los datos se publican bajo la licencia abierta **ODbL**. Este bien común digital mundial es utilizado por miles de aplicaciones y constituye hoy una referencia para muchos usos.

Cuando estos datos se vuelven críticos para un servicio, surge una pregunta: **¿cómo controlar la calidad de los datos provenientes de OpenStreetMap?**

OpenStreetMap evoluciona constantemente. Cada día, millones de modificaciones son realizadas por una comunidad mundial de contribuidores: voluntarios, instituciones públicas y empresas. Esta dinámica es una fortaleza, pero los errores, generalmente involuntarios, pueden afectar sus servicios:

- **Ediciones accidentales** — Un contribuidor bien intencionado mueve un punto de interés por error
- **Inconsistencias** — Tags incorrectos o geometrías inválidas
- **Eliminaciones no deseadas** — Objetos eliminados sin razón válida
- **Vandalismo** — Modificaciones maliciosas que corrompen los datos

Las herramientas clásicas de replicación de datos OSM no incluyen un filtro de calidad. Operan sobre datos ya replicados, o al costo de bloquear la replicación. Clearance resuelve este problema estableciendo un control de calidad **durante** la replicación.

## Principio de funcionamiento

Clearance nunca modifica OpenStreetMap. Los datos problemáticos simplemente se retienen, y las correcciones se realizan directamente en OSM en la fuente.

```
OpenStreetMap → Replicación → Clearance → Sus aplicaciones
                                 ↓
                          Cambios conformes → Pasan automáticamente
                          Cambios sospechosos → Revisión humana
```

1. **Replicación** — Clearance se conecta al flujo de replicación OSM
2. **Análisis** — Cada cambio se evalúa según las reglas de validación
3. **Filtrado** — Los cambios conformes se integran automáticamente
4. **Revisión** — Los cambios sospechosos se presentan para validación humana

Así obtiene una versión actualizada de los datos OpenStreetMap, fiable para sus usos y adaptada a sus restricciones de calidad, mientras sigue contribuyendo al bien común OpenStreetMap.
