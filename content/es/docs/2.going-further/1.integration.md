# Integración

Clearance produce salidas en formatos estándar del ecosistema OpenStreetMap, facilitando la integración con sus herramientas existentes.

## Salidas estándar

### Archivos de diferencias OSM

Clearance produce archivos de diferencias (`.osc`) en formato estándar OSM. Estos archivos pueden ser consumidos por cualquier herramienta compatible con el formato de replicación OSM:

- **Imposm** — Importación en PostgreSQL/PostGIS
- **osm2pgsql** — Importación en PostgreSQL con esquema personalizado
- **Osmosis** — Procesamiento de datos OSM

### Base de datos validada

Clearance mantiene una base de datos que solo contiene los cambios validados. Esta base puede consultarse directamente.

## Proxy Overpass API

Clearance ofrece un **proxy Overpass API** transparente. Sus aplicaciones que ya consultan la Overpass API pueden apuntar al proxy de Clearance sin modificar el código.

El proxy:

- Intercepta las consultas Overpass
- Las redirige a la base de datos validada por Clearance
- Devuelve los resultados en el formato estándar de Overpass

**Ventaja**: sus aplicaciones obtienen datos filtrados por Clearance sin ninguna modificación del lado del cliente.

## Arquitectura de integración

```
OpenStreetMap
      ↓
  Replicación
      ↓
  Clearance ──→ Archivos .osc ──→ osm2pgsql / Imposm
      ↓
  Proxy Overpass ──→ Sus aplicaciones
```
