# Hoja de ruta

Clearance está en desarrollo activo. Estas son las grandes orientaciones para las próximas versiones.

## Financiación NLNet

El desarrollo de Clearance cuenta con el apoyo de **NLNet**, en el marco del programa NGI (Next Generation Internet). Esta financiación permite acelerar el desarrollo de funcionalidades clave.

## Fuentes de datos externas

**Objetivo**: Enriquecer el análisis de Clearance integrando fuentes de datos externas para comparación y validación.

- Comparación con bases de datos de referencia (catastro, bases de direcciones, etc.)
- Detección de regresiones respecto a datos oficiales
- Validación cruzada con otras fuentes abiertas

## Sistema de reputación

**Objetivo**: Refinar el filtrado teniendo en cuenta el historial y la fiabilidad de los contribuidores.

- Puntuación de confianza por contribuidor basada en el historial
- Umbrales de validación adaptativos según la reputación
- Aprendizaje progresivo a partir de las decisiones de validación

## Mejoras de la interfaz

- Panel de control con estadísticas de validación
- Visualización cartográfica mejorada de los clusters LoCha
- Notificaciones y alertas configurables

## Contribuir

Clearance es open source. Las contribuciones son bienvenidas en el [repositorio GitHub](https://github.com/teritorio/clearance).

- Reportar un error o proponer una mejora a través de issues
- Enviar un pull request
- Participar en las discusiones
